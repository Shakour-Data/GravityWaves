Frontend Data Requirements from Backend Endpoints

This document summarizes the data and API endpoints that each frontend page consumes or requires from the backend.

1. Index Page
- Data: List of analyses with title, short_desc, long_desc
- Endpoint: None (static or initial data passed via template)

2. Login and Registration Pages
- Data: User authentication forms and validation
- Endpoint: /login, /register, /password_reset (POST)

3. Market Dashboard Page
- Data: Market overview, user-specific data
- Endpoint: /api/market_status_analysis, /api/market_data

4. Forecast Analysis Page
- Data: Forecast results, model performance, feature importances
- Endpoint: /api/forecast_analysis

5. Price History Analysis Page
- Data: Historical price data, export files
- Endpoint: /api/market_data, /api/assistant (for chatbot price queries)

6. Indicator Analysis Page
- Data: Indicator calculation results
- Endpoint: /api/indicator_analysis

7. Optimization Results Page
- Data: Optimization parameters and results
- Endpoint: /api/optimization_results

8. Comparative Results Page
- Data: Comparative analysis of multiple tickers
- Endpoint: /api/comparative_results

9. Custom Analysis Page
- Data: Custom analysis parameters and results
- Endpoint: /api/custom_analysis

10. Candlestick Chart Page
- Data: Candlestick data for chart rendering
- Endpoint: /api/market_data

11. Chatbot UI
- Data: Chat messages and replies
- Endpoint: /api/assistant, /api/chatbot

12. Trading Signals Backtesting
- Data: Backtesting parameters and results
- Endpoint: /api/trading_signals_backtesting

This document serves as a reference for frontend developers to understand backend data dependencies and integration points.
